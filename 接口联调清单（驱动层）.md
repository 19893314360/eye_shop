# 接口联调清单（驱动层）

## 1. 文档目的
用于前后端联调阶段对齐接口路径、请求字段、返回字段与错误处理口径。  
当前驱动层已兼容 `snake_case` 与 `camelCase` 两种命名。

## 2. 联调优先顺序
1. `auth`：登录与用户上下文
2. `member`：会员新增与查询
3. `order`：开单、收款、取件
4. `appointment`：预约创建与到店
5. `after-sale`：售后申请、回访、复查
6. `sales-return/sales-exchange`：销售退换货
7. `inventory/inventory-check`：库存与盘点
8. `vision`：验光记录
9. `purchase`：采购记录
10. `settings`：系统配置
11. `finance`：汇总核对（基于订单流）

## 3. 接口清单（第一批核心）

### 3.1 认证
- 路径：`POST /auth/login`
  - 请求：`code`、`desired_role`（兼容 `desiredRole`）
  - 响应：`token`、`refresh_token`（兼容 `refreshToken`）、`expires_at`（兼容 `expiresAt`）
- 路径：`GET /auth/profile`
  - 响应：`user_id`、`user_name`、`role`、`permissions`、`store_id`、`store_name`、`mobile`
  - 兼容：`userId/userName/storeId/storeName`

### 3.2 会员
- 路径：`POST /members`
  - 请求：`member_name`、`phone`、`gender`、`birthday`、`note`
  - 兼容：`name/mobile`
  - 响应：`id(member_id)`、`name(member_name)`、`mobile(phone)`、`created_at(createdAt)`...
- 路径：`GET /members`
  - 查询：`keyword`
  - 响应：会员数组

### 3.3 订单
- 路径：`POST /orders`
  - 请求：`member_id`、`order_type`、`item_name`、`quantity`、`unit_price`、`note`
  - 兼容：`memberId/orderType/itemName/unitPrice`
- 路径：`GET /orders`
  - 查询：`status`、`keyword`
- 路径：`POST /orders/{id}/pay`
  - 请求：`pay_channel`、`paid_amount`
  - 兼容：`payChannel/paidAmount`
- 路径：`POST /orders/{id}/deliver`
  - 请求：空体

## 5. 错误码约定建议
1. `0`：成功
2. `400`：参数错误
3. `401`：未登录或 token 失效（前端会清理会话）
4. `403`：权限不足
5. `404`：资源不存在
6. `409`：状态冲突（如已收款订单再次收款）
7. `500`：服务端异常

## 6. 联调配置
1. 页面路径：`系统设置 -> 系统设置 Tab -> 联调配置`
2. 可配置项：
   - `使用 Mock 数据`（开/关）
   - `API Base URL`
   - `请求字段风格`（`snake_case / camelCase / 双写兼容`）
3. 自检按钮：
   - `联调自检`：验证读链路（auth/member/order）
   - `主链路联调（写入）`：验证写链路（member/order/pay/deliver）
4. 切换真实后端：关闭 `Mock` 并填写 `API Base URL`
5. 当前本地后端已切到 `MySQL 8`，默认库为 `127.0.0.1:3306 / yanjing_dev`
6. 初始化数据库可执行：`npm.cmd run backend:db:init`
