## 3. 已完整接入真实后端的模块

### 3.1 认证
1. 登录与用户上下文已打通：`wx.login -> /auth/login -> /auth/profile`
2. 角色切换会重新登录并刷新权限上下文

### 3.2 会员
1. `member-create`：可创建会员
2. `member-query`：可按关键字查询会员

### 3.3 订单主链路
1. `order-create`：可开单
2. `order-list`：可查单
3. `payment`：可收款
4. `delivery`：可取件
5. `settings -> 联调配置`：可执行 `auth -> member -> order -> payment -> delivery` 主链路写入校验

### 3.4 预约服务
1. `appointment`：可读取预约列表
2. `appointment`：可创建预约
3. `appointment`：可标记到店

### 3.5 售后申请与追踪
1. `after-sale-apply`：可发起售后申请
2. `after-sale`：可读取售后申请列表与处理记录
3. `after-sale`：可登记回访
4. `after-sale`：可处理复查

### 3.6 销售退货与换货
1. `sales-return`：可读取退货记录
2. `sales-return`：可创建退货记录
3. `sales-exchange`：可读取换货记录
4. `sales-exchange`：可创建换货记录

### 3.7 验光记录
1. `optometry-create`：可新增验光记录
2. `vision-profile`：可查看“验光记录”列表

### 3.8 采购记录
1. `purchase`：可新增采购记录
2. `purchase`：可查看采购记录列表

### 3.9 系统设置
1. `settings`：可读取系统设置
2. `settings`：可保存系统设置

### 3.10 库存与盘点
1. `inventory`：可读取库存列表
2. `inventory`：可读取库存追踪日志
3. `inventory`：可手工调整库存
4. `inventory-check`：可创建盘点任务
5. `inventory-check`：可读取盘点项列表
6. `inventory-check`：可提交单项盘点结果
7. `inventory-check`：可批量提交盘点结果
8. `inventory-check`：可处理差异并入账

## 4. 可用但不够完整的模块

### 4.1 订单详情
1. `order-detail` 当前不是独立详情接口
2. 实现方式是先调用 `listOrders()`，再在前端本地按 `orderId` 查找
3. 后续建议补 `GET /orders/:id`

### 4.2 财务汇总
1. `finance-summary` 页面当前基于订单列表在前端计算汇总
2. 本地后端虽然已实现 `GET /finance/summary`，但页面尚未直接使用

### 4.3 视力档案
1. `vision-profile` 的“验光记录”页签已接真实后端
2. “生物测量”“视力筛查”两个页签仍是静态展示壳子，未接真实数据源
