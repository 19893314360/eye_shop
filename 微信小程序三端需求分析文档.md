## 2. 项目背景与目标
当前业务形态属于典型的门店零售+会员+视光服务模式，且已形成较完整的线下业务链路（验光、开单、加工、库存、取件、售后、复购、视训）。

建设微信小程序三端的目标：
1. 客户端：提升客户自助服务能力，减少门店重复沟通成本。
2. 销售员端：提升开单与会员服务效率，形成标准化流程。
3. 管理者端：实现经营可视化、流程可控化、权限可管理化。

## 7. 详细需求（按模块）

### 7.1 账户与权限
- FR-001：支持微信授权登录，首次登录需绑定手机号。
- FR-002：采用 RBAC（角色+门店+数据范围）权限模型。
- FR-003：支持一人多角色切换（如店长兼销售）。
- FR-004：关键行为记录操作日志（登录、改价、退款、盘点、审批）。

### 7.2 销售开单与会员
- FR-010：支持普通开单与验光开单两种流程。
- FR-011：开单时自动读取会员信息、历史配镜记录与推荐参数。
- FR-012：支持多支付方式（微信支付、余额、组合支付）。
- FR-013：订单状态最少包含：待支付、待加工、待取件、已完成、售后中。
- FR-014：支持取件提醒（订阅消息+站内消息）。
- FR-015：支持会员充值、积分累计、积分抵扣与兑换规则配置。

### 7.3 视训服务
- FR-020：支持视训方案创建、发布、执行记录、核销。
- FR-021：支持设备租借（出借、归还、逾期提醒）。
- FR-022：支持视训提醒（按计划自动推送）。
- FR-023：支持客户查看个人训练进度与历史记录。

### 7.4 库存、加工与采购
- FR-030：支持库存查询、库存不足、库存超限、库存追踪。
- FR-031：支持调拨出库与调拨收货双向流程。
- FR-032：支持库存盘点（盘前锁定、盘中记录、盘后差异处理）。
- FR-033：支持开始加工、加工进度更新、完工入库。
- FR-034：支持采购订单、采购入库、采购退货闭环。
- FR-035：支持镜片价目册与销售价格规则维护。

### 7.5 财务与报表
- FR-040：提供经营状况看板（总览、详情、收款、欠款）。
- FR-041：支持按“今日/本月”切换统计口径。
- FR-042：支持销售员业绩排行榜（可配置显示前 N 名）。
- FR-043：支持门店维度、人员维度、品类维度钻取分析。
- FR-044：关键指标与订单数据实时联动，允许 T+0 查询。

### 7.6 系统设置与设备
- FR-050：支持打印模板管理（小票、配镜单、取件单）。
- FR-051：支持收款商户号、费率展示与配置（管理者权限）。
- FR-052：支持绑定收款盒子及设备在线状态查看。
- FR-053：提供软件更新日志、设备支持文档、教学视频入口。
- FR-054：支持公告发布与门店公告阅读回执。

## 8. 关键业务流程

### 8.1 销售闭环流程
1. 客户预约/到店。
2. 销售员新增会员或识别老会员。
3. 验光并录入验光数据。
4. 开单（普通/验光），计算优惠与应收。
5. 支付收款，订单进入待加工。
6. 加工完成后触发取件提醒。
7. 客户取件核销，订单完成。
8. 触发售后回访与复购推荐。

### 8.2 视训闭环流程
1. 制定视训方案。
2. 分配训练周期与提醒频次。
3. 客户执行训练并记录。
4. 销售员核销课程/训练次数。
5. 管理者查看完成率与转化率。

### 8.3 库存采购闭环流程
1. 系统识别库存不足/超限。
2. 生成采购建议或调拨建议。
3. 管理者审批采购单。
4. 入库并更新库存成本。
5. 报表同步更新周转率和缺货率。

## 10. 外部接口与技术边界
1. 微信能力：微信登录、手机号绑定、支付、订阅消息、小程序码。
2. 硬件能力：蓝牙/网络打印机、收款盒子、扫码枪（如有）。
3. 第三方能力：短信通道（可选）、对象存储、日志告警平台。
4. 与存量系统关系：如已有 ERP/POS，需明确主数据归属与同步策略（门店、商品、会员、库存）。

## 11. 非功能需求
- 性能：核心页面首屏 <= 2 秒；关键查询 API P95 <= 500ms。
- 可用性：服务可用性 >= 99.9%；支付链路具备降级与重试。
- 安全：敏感信息脱敏展示；HTTPS 全链路；关键操作二次确认。
- 合规：符合个人信息保护要求，支持用户授权、撤回与数据最小化采集。
- 可观测：提供日志、指标、告警，支持按订单号全链路追踪。
- 可扩展：支持多门店、多角色、多套餐策略扩展。

## 12. 分期上线建议
| 阶段 | 目标 | 范围 |
|---|---|---|
| MVP（6-8周） | 跑通交易闭环 | 登录、会员、验光开单、支付、取件提醒、基础报表 |
| V1.1（4-6周） | 完善运营能力 | 会员充值积分、售后回访、视训方案/核销、库存预警 |
| V1.2（4-6周） | 管理精细化 | 调拨盘点、采购闭环、排行榜、系统配置、审计能力 |

## 13. 验收标准（关键）
1. 三端可按角色登录，菜单与数据权限正确隔离。
2. 从“预约/到店”到“开单-支付-加工-取件”流程可完整闭环。
3. 库存流水与订单流水可对账，差异可追溯到操作人。
4. 统计报表与订单明细一致，支持今日与本月切换。
5. 消息提醒（取件、复查、视训）可配置且发送可追踪。

## 14. 风险与待确认项
1. 待确认：是否需要一个小程序内切换三端，还是三个独立小程序。
2. 待确认：是否存在存量 ERP/POS，若有需定义双向同步规则。
3. 待确认：支付场景是否包含押金、分期、退款原路返回等复杂流程。
4. 待确认：是否需要总部-门店两级组织与审批链。
5. 风险：如果权限模型定义过晚，后续返工成本高。
6. 风险：库存与财务口径未统一会导致报表可信度下降。
